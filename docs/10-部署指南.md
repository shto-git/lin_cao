# 部署指南

## 一、MVP 阶段部署

### 1.1 环境要求

- Windows 10/11 或 Linux
- Python 3.10+
- 4GB+ 可用内存
- RAGFlow 实例（本地或远程）

### 1.2 本地启动

```powershell
# 克隆项目
git clone <repository-url>
cd lin_cao

# 创建虚拟环境
python -m venv venv
venv\Scriptsctivate

# 安装依赖
pip install python-docx

# 运行 CLI
python -m lin_cao_planner.cli samples\demo_project.json --out dist\demo

# 运行测试
python -m unittest discover -s tests -v
```

### 1.3 RAGFlow 配置

1. 启动 RAGFlow（Docker 方式参考官方文档）
2. 获取 API Key：RAGFlow 管理后台 → API Key
3. 配置环境变量：

```powershell
$env:RAGFLOW_BASE_URL="http://localhost:9380"
$env:RAGFLOW_API_KEY="your-api-key"
```

## 二、生产化部署

### 2.1 架构

```text
                    ┌─────────────────┐
                    │   Nginx / ALB   │
                    └────────┬────────┘
                             │
              ┌──────────────┼──────────────┐
              │              │              │
     ┌────────▼───────┐ ┌───▼────┐ ┌──────▼───────┐
     │  FastAPI 服务   │ │ Redis  │ │  Celery      │
     │  (业务编排)     │ │ 缓存   │ │  任务队列    │
     └────────┬───────┘ └───┬────┘ └──────┬───────┘
              │             │              │
     ┌────────▼─────────────▼──────────────▼───────┐
     │              PostgreSQL                      │
     └─────────────────────┬───────────────────────┘
                           │
                  ┌────────▼────────┐
                  │   RAGFlow       │
                  │  (独立部署)     │
                  └─────────────────┘
```

### 2.2 Docker Compose 示例

```yaml
version: "3.8"

services:
  api:
    build: .
    ports:
      - "8000:8000"
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/lin_cao
      - REDIS_URL=redis://redis:6379/0
      - RAGFLOW_BASE_URL=http://ragflow:9380
      - RAGFLOW_API_KEY=${RAGFLOW_API_KEY}
    depends_on:
      - db
      - redis
      - ragflow

  worker:
    build: .
    command: celery -A lin_cao_planner.tasks worker --loglevel=info
    environment:
      - DATABASE_URL=postgresql://user:pass@db:5432/lin_cao
      - REDIS_URL=redis://redis:6379/0
      - RAGFLOW_BASE_URL=http://ragflow:9380
      - RAGFLOW_API_KEY=${RAGFLOW_API_KEY}
    depends_on:
      - db
      - redis

  db:
    image: postgres:16
    environment:
      POSTGRES_DB: lin_cao
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
    volumes:
      - pgdata:/var/lib/postgresql/data

  redis:
    image: redis:7-alpine

  ragflow:
    image: infiniflow/ragflow:latest
    ports:
      - "9380:9380"
    # RAGFlow 具体配置参考官方文档

volumes:
  pgdata:
```

### 2.3 环境变量

| 变量 | 说明 | 默认值 |
|------|------|--------|
| DATABASE_URL | 数据库连接串 | sqlite:///./lin_cao.db |
| REDIS_URL | Redis 连接串 | redis://localhost:6379/0 |
| RAGFLOW_BASE_URL | RAGFlow 地址 | http://localhost:9380 |
| RAGFLOW_API_KEY | RAGFlow API Key | (必填) |
| LOG_LEVEL | 日志级别 | INFO |
| MAX_WORKERS | 最大并发任务数 | 4 |
| OUTPUT_DIR | 输出目录 | ./output |

## 三、私有化部署注意事项

### 3.1 模型部署

- 大模型建议本地部署（如通过 Ollama、vLLM）
- 敏感数据不出内网
- 模型调用需记录审计日志

### 3.2 数据安全

- 项目数据按租户隔离
- 资料上传需病毒扫描
- 导出文件需水印或标识
- 操作日志保留 6 个月以上

### 3.3 备份策略

- 数据库每日全量备份
- 导出文件按需保留
- RAGFlow 数据集定期快照
