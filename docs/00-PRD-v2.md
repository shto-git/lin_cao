## 一、需求痛点分析

### 1.1 核心痛点（重新聚焦）

经梳理，林草规划文本编制的真实痛点按优先级排列：

| # | 痛点 | 严重程度 | 当前解决方式 | 问题 |
|---|------|---------|------------|------|
| 1 | **资料查找耗时** | 🔥严重 | 人工翻找文件/网络搜索 | 一个规划可能要查 50+ 份文件，半天就过去了 |
| 2 | **数据口径不一致** | 🔥严重 | 人工核对 | 同一指标在不同章节出现不同值，直到被专家发现 |
| 3 | **专业表述不规范** | 中等 | 老编制人员口口相传 | 新人写的规划用词不标准，出现禁用表述 |
| 4 | **改一处牵全身** | 中等 | 人工全文搜索替换 | 改一个数据/政策，要全文排查 |
| 5 | **图表与正文脱节** | 中等 | 先写文本再补图 | 文字改了图没改，或图改了文字没更新 |

### 1.2 去掉的原需求（V1 的问题）

V1 需求存在以下问题，需要裁剪：

| V1 需求 | 问题 | 决策 |
|---------|------|------|
| "5万-10万字智能生成" | ❌ 不现实。模型一次性生成 5 万字不可控，质量堪忧 | → 改为"章节级生成，单章 3000-8000 字" |
| "10 种规划类型全覆盖" | ❌ 贪多不嚼不烂 | → 先做"林业发展规划"一种，跑通后再扩展 |
| "Word/PDF/HTML 多格式导出" | ❌ MVP 阶段多格式是过度设计 | → MVP 只出 Markdown，Phase 2 做 Word |
| "在线多人协同编辑" | ❌ 完全不在 MVP 范围 | → 移除 |
| "GIS 空间图件自动制图" | ❌ 需要专业 GIS 引擎，不适合 MVP | → 只保留图表说明文本生成 |
| "多版本大纲对比" | 🔥 伪需求，实际工作流不会用到 | → 移除 |
| "基于 GIS 生成图件说明" | ❌ 依赖空间数据导入，太复杂 | → 移除 |

### 1.3 真正的用户价值

**核心价值主张**：让编制人员在 1 小时内完成过去需要 3-5 天的初稿编写工作。

**不是替代人**，而是替代最耗时的"从零写初稿"过程。编制人员基于系统输出的初稿进行修改完善，而非从零开始。

## 二、MVP 范围（MoSCoW 优先级）

### MUST HAVE（MVP 必做）

| # | 功能 | 用户故事 | 验收标准 |
|---|------|---------|---------|
| M1 | 项目管理 | 作为编制人员，我希望能创建一个规划项目并填写基本信息（名称/区域/期限/类型/字数） | 能成功创建项目，信息保存到数据库 |
| M2 | 规划模板 | 作为编制人员，我希望能按"林业发展规划"类型自动生成标准大纲 | 生成包含 9 章的标准大纲，字数按比例分配 |
| M3 | 资料入库 | 作为编制人员，我希望能上传政策/标准/统计数据等资料，并自动解析建库 | 上传 PDF/Word 后能成功检索出相关段落 |
| M4 | 章节检索任务 | 作为编制人员，我希望能看到每章的检索计划和候选依据 | 每章有 3-5 条检索问题，能返回可追溯 chunk |
| M5 | 章节草稿生成 | 作为编制人员，我希望能逐章生成草稿，带引用来源 | 单章输出 3000-8000 字，标注引用来源，可导出 Markdown |
| M6 | 基础质检 | 作为审核人员，我希望能看到质检报告 | 能发现无来源数据、禁用词、指标冲突、缺章缺节 |
| M7 | Markdown 导出 | 作为编制人员，我希望能导出初稿为 Markdown | 导出的 Markdown 包含完整大纲、章节内容、引用、质检报告 |

### SHOULD HAVE（Phase 2）

| # | 功能 | 理由 |
|---|------|------|
| S1 | Word 导出 | 业务最终交付物是 Word，但 MVP 先用 Markdown 验证流程 |
| S2 | 章节扩写/压缩/重写 | 基于质检反馈修改章节的能力 |
| S3 | 指标一致性检查 | 全文指标索引 + 冲突检测 |
| S4 | 富文本编辑器 | 在线编辑草稿，替代纯 Markdown |
| S5 | 更多规划类型模板 | 湿地、草原、自然保护地等 |

### COULD HAVE（Phase 3）

| # | 功能 | 理由 |
|---|------|------|
| C1 | 统计图表自动生成 | 需要结构化数据支撑，MVP 先不做 |
| C2 | 审查意见闭环 | 依赖协同工作流，先不做 |
| C3 | PDF 导出 | 优先级低于 Word |

### WON'T HAVE（明确不做）

- 在线多人协同编辑
- GIS 自动制图
- 移动端
- 多租户权限
- 类 Word 排版器（MVP 阶段）

## 三、用户故事详述

### 3.1 创建项目

```
As a 编制人员
I want 创建一个新的规划项目，填写名称、区域、期限、规划类型、目标字数
So that 系统能为我生成对应类型的标准大纲
```

验收标准：
1. 输入"某县林业发展规划"、区域"XX县"、期限"2026-2030"、类型"林业发展规划"、字数 50000
2. 系统创建项目并自动生成大纲
3. 项目列表能看到刚创建的项目

### 3.2 上传资料并建库

```
As a 编制人员
I want 上传规划相关的政策文件、统计数据、历史规划等资料
So that 系统能基于这些资料为我生成有据可查的章节内容
```

验收标准：
1. 支持上传 PDF、Word、Excel 文件
2. 上传后自动解析并加入 RAGFlow 知识库
3. 检索"森林覆盖率"能返回相关段落并标注来源文件名

### 3.3 生成章节草稿

```
As a 编制人员
I want 逐章生成规划草稿，每章附带引用来源
So that 我能在初稿基础上修改完善，而不是从零开始写
```

验收标准：
1. 点击"生成大纲"后，每章自动生成检索问题
2. 点击"生成草稿"后，该章输出 3000-8000 字
3. 草稿中标注引用来源（如"来源：XX县森林资源调查报告（2024）"）
4. 缺资料时显示风险提示而非编造内容

### 3.4 质检报告

```
As a 审核人员
I want 看到一份自动生成的质检报告
So that 我能快速定位文本中的问题，不用逐字检查
```

验收标准：
1. 报告分类：错误/警告/提示
2. 错误：无来源数据、禁用词、指标冲突
3. 警告：术语不规范、编号缺失
4. 每条问题标注位置（章节 + 段落）

## 四、非功能性需求

### 4.1 性能
- 大纲生成：< 5 秒
- 章节草稿生成：< 30 秒（调用 LLM）
- 资料检索：< 3 秒
- 质检报告：< 10 秒

### 4.2 可用性
- 系统可用性 > 99%（本地部署）
- 数据不丢失（SQLite 定期备份）

### 4.3 安全
- 数据本地存储，不上传外部
- API Key 本地加密存储

### 4.4 兼容性
- 支持 Chrome/Edge 浏览器
- 支持 Windows 10/11

## 五、风险识别

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| LLM 生成内容质量差 | 用户不信任系统 | 中 | 缺资料时强制降级为风险提示，不编造 |
| RAGFlow 解析失败 | 检索不到依据 | 中 | 解析结果可预览，支持手动修正 |
| 数据口径冲突 | 同一指标不同值 | 高 | 全文指标索引 + 一致性检查 |
| 用户期望过高 | 认为能全自动生成 | 高 | 明确宣传"辅助生成 + 人工审核"定位 |
| 模型 API 费用 | 成本不可控 | 低 | 设置调用上限，优先用免费模型 |

## 八、术语表

| 术语 | 定义 |
|------|------|
| RAGFlow | 开源 RAG 引擎，用于文档解析和知识检索 |
| Chunk | 文档被切分后的最小段落单元 |
| 检索计划 | 每章对应的检索问题列表 |
| 证据包 | 检索到的相关段落及其来源信息 |
| 质检引擎 | 检查文本中数据/术语/逻辑问题的规则系统 |
