# 数据模型与数据库设计

本文档定义系统的持久化数据对象。MVP 阶段使用 SQLite，生产化阶段迁移到 PostgreSQL。

---

## 一、实体关系概览

```text
Project (项目)
  ├── Document (资料)
  ├── OutlineNode (大纲节点)
  │     └── SectionTask (章节任务)
  │           ├── Evidence (证据)
  │           └── ChapterDraft (章节草稿)
  │                 └── DraftVersion (草稿版本)
  ├── QualityReport (质检报告)
  │     └── QualityFinding (质检发现)
  └── ExportTask (导出任务)
```

---

## 二、表结构

### 2.1 projects（项目表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 项目唯一标识 |
| name | VARCHAR(200) | NOT NULL | 规划名称 |
| region | VARCHAR(100) | NOT NULL | 规划区域 |
| period | VARCHAR(50) | NOT NULL | 规划期限 |
| level | VARCHAR(20) | | 规划层级：国家级/省级/市级/县级 |
| planning_type | VARCHAR(50) | NOT NULL | 规划类型 |
| target_words | INT | DEFAULT 50000 | 目标字数 |
| owner_department | VARCHAR(200) | | 主管部门 |
| drafting_unit | VARCHAR(200) | | 编制单位 |
| status | VARCHAR(20) | DEFAULT 'active' | 状态：active/archived/deleted |
| ragflow_dataset_id | VARCHAR(100) | | 关联 RAGFlow 数据集 ID |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |

### 2.2 documents（资料表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 资料唯一标识 |
| project_id | UUID | FK → projects.id | 所属项目 |
| file_name | VARCHAR(500) | NOT NULL | 文件名 |
| file_path | VARCHAR(1000) | | 存储路径 |
| file_type | VARCHAR(20) | | 文件类型：pdf/docx/xlsx/md/txt |
| file_size | INT | | 文件大小（字节） |
| source_unit | VARCHAR(200) | | 来源单位 |
| publish_year | INT | | 发布年份 |
| applicable_region | VARCHAR(200) | | 适用区域 |
| applicable_planning_types | VARCHAR(500) | | 适用规划类型（JSON 数组） |
| is_formal_reference | BOOLEAN | DEFAULT true | 是否可作为正式依据 |
| category | VARCHAR(50) | | 资料分类：policy/standard/data/case/other |
| ragflow_document_id | VARCHAR(100) | | RAGFlow 文档 ID |
| parse_status | VARCHAR(20) | DEFAULT 'pending' | 解析状态：pending/parsing/completed/failed |
| chunk_count | INT | | 解析后 chunk 数量 |
| description | TEXT | | 资料摘要 |
| created_at | TIMESTAMP | DEFAULT NOW() | 上传时间 |

### 2.3 outline_nodes（大纲节点表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | VARCHAR(20) | PK | 节点编号，如 "1"、"1.1"、"2.3" |
| project_id | UUID | FK → projects.id | 所属项目 |
| parent_id | VARCHAR(20) | FK → outline_nodes.id | 父节点编号 |
| title | VARCHAR(200) | NOT NULL | 节点标题 |
| level | INT | NOT NULL | 层级：0=根，1=章，2=节 |
| target_words | INT | | 目标字数 |
| sort_order | INT | | 同级排序 |
| requirements | TEXT | | 写作要求（JSON 数组） |
| required_evidence_types | VARCHAR(500) | | 必引资料类型（JSON 数组） |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |

### 2.4 section_tasks（章节任务表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 任务唯一标识 |
| project_id | UUID | FK → projects.id | 所属项目 |
| outline_id | VARCHAR(20) | FK → outline_nodes.id | 关联大纲节点 |
| title_path | VARCHAR(500) | | 标题路径（JSON 数组） |
| target_words | INT | | 目标字数 |
| retrieval_queries | TEXT | | 检索问题（JSON 数组） |
| required_evidence_types | VARCHAR(500) | | 必引资料类型 |
| writing_constraints | TEXT | | 写作约束（JSON 数组） |
| status | VARCHAR(20) | DEFAULT 'pending' | 状态：pending/retrieving/drafting/reviewing/completed |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |

### 2.5 evidences（证据表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 证据唯一标识 |
| project_id | UUID | FK → projects.id | 所属项目 |
| section_task_id | UUID | FK → section_tasks.id | 关联章节任务 |
| title | VARCHAR(500) | | 证据标题 |
| source_type | VARCHAR(50) | | 来源类型：policy/standard/data/case |
| source_name | VARCHAR(500) | | 来源文件名 |
| quote | TEXT | | 引用原文 |
| similarity | FLOAT | | 检索相似度 |
| chunk_id | VARCHAR(100) | | RAGFlow chunk ID |
| metadata | JSONB | | 元数据（页码、位置等） |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |

### 2.6 chapter_drafts（章节草稿表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 草稿唯一标识 |
| project_id | UUID | FK → projects.id | 所属项目 |
| outline_id | VARCHAR(20) | FK → outline_nodes.id | 关联大纲节点 |
| title | VARCHAR(200) | | 章节标题 |
| content | TEXT | | 草稿内容 |
| word_count | INT | | 实际字数 |
| evidence_ids | TEXT | | 引用证据 ID（JSON 数组） |
| status | VARCHAR(20) | DEFAULT 'draft' | 状态：draft/reviewing/approved/rejected |
| version | INT | DEFAULT 1 | 版本号 |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT NOW() | 更新时间 |

### 2.7 draft_versions（草稿版本表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 版本唯一标识 |
| chapter_draft_id | UUID | FK → chapter_drafts.id | 关联草稿 |
| version | INT | NOT NULL | 版本号 |
| content | TEXT | | 版本内容 |
| change_summary | VARCHAR(500) | | 修改摘要 |
| created_by | VARCHAR(100) | | 创建人 |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |

### 2.8 quality_reports（质检报告表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 报告唯一标识 |
| project_id | UUID | FK → projects.id | 所属项目 |
| status | VARCHAR(20) | DEFAULT 'completed' | 状态 |
| total_findings | INT | DEFAULT 0 | 发现问题总数 |
| error_count | INT | DEFAULT 0 | 错误数量 |
| warning_count | INT | DEFAULT 0 | 警告数量 |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |

### 2.9 quality_findings（质检发现表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 发现唯一标识 |
| report_id | UUID | FK → quality_reports.id | 所属报告 |
| severity | VARCHAR(10) | NOT NULL | 严重程度：error/warning/info |
| code | VARCHAR(50) | NOT NULL | 问题编码 |
| message | TEXT | NOT NULL | 问题描述 |
| location | VARCHAR(100) | | 位置（大纲节点 ID） |
| suggestion | TEXT | | 修改建议 |
| status | VARCHAR(20) | DEFAULT 'open' | 状态：open/resolved/ignored |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |

### 2.10 export_tasks（导出任务表）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PK | 任务唯一标识 |
| project_id | UUID | FK → projects.id | 所属项目 |
| format | VARCHAR(20) | NOT NULL | 导出格式：markdown/word/pdf/html |
| file_path | VARCHAR(1000) | | 导出文件路径 |
| status | VARCHAR(20) | DEFAULT 'pending' | 状态：pending/processing/completed/failed |
| error_message | TEXT | | 错误信息 |
| created_at | TIMESTAMP | DEFAULT NOW() | 创建时间 |
| completed_at | TIMESTAMP | | 完成时间 |

## 三、索引设计

```sql
-- 常用查询索引
CREATE INDEX idx_documents_project ON documents(project_id);
CREATE INDEX idx_outline_nodes_project ON outline_nodes(project_id);
CREATE INDEX idx_section_tasks_project ON section_tasks(project_id);
CREATE INDEX idx_section_tasks_outline ON section_tasks(outline_id);
CREATE INDEX idx_evidences_task ON evidences(section_task_id);
CREATE INDEX idx_chapter_drafts_project ON chapter_drafts(project_id);
CREATE INDEX idx_quality_findings_report ON quality_findings(report_id);
CREATE INDEX idx_export_tasks_project ON export_tasks(project_id);

-- 状态筛选索引
CREATE INDEX idx_section_tasks_status ON section_tasks(status);
CREATE INDEX idx_chapter_drafts_status ON chapter_drafts(status);
CREATE INDEX idx_quality_findings_severity ON quality_findings(severity);
```

## 四、SQLite 建表语句（MVP 阶段）

```sql
CREATE TABLE projects (
    id TEXT PRIMARY KEY,
    name TEXT NOT NULL,
    region TEXT NOT NULL,
    period TEXT NOT NULL,
    level TEXT,
    planning_type TEXT NOT NULL,
    target_words INTEGER DEFAULT 50000,
    owner_department TEXT,
    drafting_unit TEXT,
    status TEXT DEFAULT 'active',
    ragflow_dataset_id TEXT,
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE documents (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL REFERENCES projects(id),
    file_name TEXT NOT NULL,
    file_path TEXT,
    file_type TEXT,
    file_size INTEGER,
    source_unit TEXT,
    publish_year INTEGER,
    applicable_region TEXT,
    applicable_planning_types TEXT,
    is_formal_reference INTEGER DEFAULT 1,
    category TEXT,
    ragflow_document_id TEXT,
    parse_status TEXT DEFAULT 'pending',
    chunk_count INTEGER,
    description TEXT,
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE outline_nodes (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL REFERENCES projects(id),
    parent_id TEXT REFERENCES outline_nodes(id),
    title TEXT NOT NULL,
    level INTEGER NOT NULL,
    target_words INTEGER,
    sort_order INTEGER,
    requirements TEXT,
    required_evidence_types TEXT,
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE section_tasks (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL REFERENCES projects(id),
    outline_id TEXT NOT NULL REFERENCES outline_nodes(id),
    title_path TEXT,
    target_words INTEGER,
    retrieval_queries TEXT,
    required_evidence_types TEXT,
    writing_constraints TEXT,
    status TEXT DEFAULT 'pending',
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE evidences (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL REFERENCES projects(id),
    section_task_id TEXT REFERENCES section_tasks(id),
    title TEXT,
    source_type TEXT,
    source_name TEXT,
    quote TEXT,
    similarity REAL,
    chunk_id TEXT,
    metadata TEXT,
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE chapter_drafts (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL REFERENCES projects(id),
    outline_id TEXT NOT NULL REFERENCES outline_nodes(id),
    title TEXT,
    content TEXT,
    word_count INTEGER,
    evidence_ids TEXT,
    status TEXT DEFAULT 'draft',
    version INTEGER DEFAULT 1,
    created_at TEXT DEFAULT (datetime('now')),
    updated_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE quality_reports (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL REFERENCES projects(id),
    status TEXT DEFAULT 'completed',
    total_findings INTEGER DEFAULT 0,
    error_count INTEGER DEFAULT 0,
    warning_count INTEGER DEFAULT 0,
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE quality_findings (
    id TEXT PRIMARY KEY,
    report_id TEXT NOT NULL REFERENCES quality_reports(id),
    severity TEXT NOT NULL,
    code TEXT NOT NULL,
    message TEXT NOT NULL,
    location TEXT,
    suggestion TEXT,
    status TEXT DEFAULT 'open',
    created_at TEXT DEFAULT (datetime('now'))
);

CREATE TABLE export_tasks (
    id TEXT PRIMARY KEY,
    project_id TEXT NOT NULL REFERENCES projects(id),
    format TEXT NOT NULL,
    file_path TEXT,
    status TEXT DEFAULT 'pending',
    error_message TEXT,
    created_at TEXT DEFAULT (datetime('now')),
    completed_at TEXT
);
```
