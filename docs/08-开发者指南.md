## 四、模块说明

### 4.1 domain.py — 数据对象

| 类 | 用途 |
|----|------|
| ProjectInfo | 项目信息，含基本信息和字数目标 |
| OutlineNode | 大纲树节点，支持多级嵌套 |
| SectionBrief | 章节检索计划 |
| Evidence | 证据/引用来源 |
| ChapterDraft | 章节草稿 |
| QualityFinding | 质检发现 |

### 4.2 outline.py — 大纲生成

核心函数：`build_default_outline(project) -> OutlineNode`

模板系统：
- `BASE_TEMPLATE`：基础通用模板
- `TYPE_OVERRIDES`：按规划类型的覆盖规则
- `_select_template()`：根据规划类型自动匹配模板
- `_distribute_words()：字数分配算法

### 4.3 evidence.py — 检索计划

核心函数：`build_retrieval_plan(project, outline) -> list[SectionBrief]`

为每个章节叶子节点生成 3-5 条检索问题，包含：
- 基础资料查询
- 政策法规查询
- 历史案例查询
- 空间数据/项目数据专项查询（按 required_evidence_types 触发）

### 4.4 validators.py — 质检引擎

核心函数：`validate_document(outline, drafts) -> list[QualityFinding]`

规则分类：
- `missing_section` — 缺少章节草稿
- `forbidden_phrase` — 不规范表述（据说/大概/有关部门等）
- `missing_evidence` — 无来源事实或数据
- `metric_conflict` — 指标口径冲突

### 4.5 ragflow_client.py — RAGFlow 适配

封装 RAGFlow REST API，当前实现：
- 创建知识库
- 触发文档解析
- 检索 chunk

待补充：
- 上传文档
- 删除文档/知识库
- Chat 接口（OpenAI-compatible）

### 4.6 renderer.py — 输出渲染

将内部数据对象转为 Markdown 格式：
- `render_outline()` → 大纲 Markdown
- `render_retrieval_plan()` → 检索计划 Markdown
- `render_quality_report()` → 质检报告 Markdown

## 五、开发规范

### 代码风格

- 遵循 PEP 8
- 类型注解必须
- 文档字符串使用 Google Style
- 提交前运行 `python -m unittest discover -s tests -v`

### 分支策略

```text
main          — 稳定发布分支
  ├── develop — 开发集成分支
  │     ├── feature/xxx  — 功能分支
  │     └── fix/xxx      — 修复分支
  └── hotfix/xxx — 紧急修复
```

### 提交规范

```
feat: 新增章节扩写功能
fix: 修复指标冲突检测中正则匹配跨行失效
docs: 补充 API 接口设计文档
refactor: 提取字数分配公共方法
test: 添加湿地规划模板测试用例
```

## 六、故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| `ModuleNotFoundError: No module named 'lin_cao_planner'` | 未在项目根目录执行或未将根目录加入 PYTHONPATH | 在 `lin_cao/` 目录下执行，或 `sys.path.insert(0, 'D:\lincao')` |
| RAGFlow 连接超时 | RAGFlow 未启动或网络不通 | 检查 RAGFlow 服务状态和 `RAGFLOW_BASE_URL` |
| 检索结果为空 | 数据集未解析或相似度阈值过高 | 确认文档已解析完成，降低 `similarity_threshold` |
| 字数分配不均 | 模板权重未归一化 | 检查 `TYPE_OVERRIDES` 中权重总和，调用 `_normalize_weights()` |
