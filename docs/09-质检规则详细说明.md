# 质检规则详细说明

本文档定义系统所有质检规则的检测逻辑、错误严重程度、修复建议和扩展方式。

---

## 一、规则体系

### 优先级定义

| 优先级 | 类别 | 说明 |
|--------|------|------|
| P0 | 事实和数据 | 直接影响成果可信度：无来源数据、指标冲突 |
| P1 | 政策和术语 | 影响专业规范性：禁用词、模糊表述、政策简称 |
| P2 | 格式和编号 | 影响可读性和一致性：章节缺失、编号连续 |
| P3 | 逻辑结构 | 影响规划质量：逻辑跳跃、内容重复 |

### 严重程度定义

| 级别 | 说明 | 处理要求 |
|------|------|----------|
| `error` | 必须修复的问题 | 成果交付前必须解决 |
| `warning` | 建议修复的问题 | 评审时应确认 |
| `info` | 提示信息 | 供参考 |

## 二、规则清单

### 2.1 来源与证据类

#### missing_section（缺少章节）

- **代码**：`missing_section`
- **优先级**：P2
- **严重程度**：`warning`
- **检测逻辑**：对比大纲叶子节点列表与已有草稿集合，找出未被覆盖的章节
- **触发条件**：`outline.leaves()` 中存在节点的 `id` 不在 `drafts` 的 `outline_id` 集合中
- **修复建议**：生成或补录该章节后再做全文质检

#### missing_evidence（无来源事实）

- **代码**：`missing_evidence`
- **优先级**：P0
- **严重程度**：`error`
- **检测逻辑**：章节草稿中出现事实性表述标记（"根据"、"统计"、"监测"等）或数字+单位组合，但未绑定任何证据 ID
- **触发条件**：`(has_fact_marker OR has_number) AND len(evidence_ids) == 0`
- **修复建议**：先从资料库检索依据并绑定 evidence_id，或改为风险提示表述
- **事实标记列表**：`根据`、`依据`、`按照`、`统计`、`监测`、`调查`、`年鉴`、`数据显示`
- **数字+单位模式**：正则 `\d+(?:\.\d+)?\s*(?:%|公顷|亩|万亩|平方公里|万元|亿元|年)`

#### evidence_source_invalid（证据来源无效）

- **代码**：待实现
- **优先级**：P0
- **严重程度**：`error`
- **检测逻辑**：引用的 evidence_id 不在证据表中，或来源资料已标记为失效
- **修复建议**：更新证据引用或补充新证据

### 2.2 禁用表述类

#### forbidden_phrase（不规范表述）

- **代码**：`forbidden_phrase`
- **优先级**：P1
- **严重程度**：`warning`
- **检测逻辑**：扫描草稿中是否包含预设禁用词/短语
- **当前禁用词表**：

| 禁用词 | 问题 | 替换建议 |
|--------|------|----------|
| 据说 | 不可核验 | 改为可核验来源，如文件名、统计年鉴或主管部门材料 |
| 大概 | 不精确 | 改为明确数字，或标注为待补充数据 |
| 有关部门 | 模糊责任 | 写明具体部门或责任单位 |
| 相关政策 | 模糊依据 | 写明政策文件全称 |

**扩展禁用词**：在 `validators.py` 的 `FORBIDDEN_PHRASES` 字典中添加即可。

### 2.3 指标一致性类

#### metric_conflict（指标口径冲突）

- **代码**：`metric_conflict`
- **优先级**：P0
- **严重程度**：`error`
- **检测逻辑**：
  1. 遍历所有草稿，用正则匹配指标名+数值
  2. 按指标名分组，将同一指标的所有数值归类
  3. 若同一指标出现不同数值且未区分年份/范围，判定为冲突
- **当前监控指标**：

| 指标名 | 匹配正则 | 单位 |
|--------|----------|------|
| 森林覆盖率 | `森林覆盖率[^\d%]{0,16}(\d+(?:\.\d+)?)\s*%` | % |
| 草原综合植被盖度 | `草原综合植被盖度[^\d%]{0,16}(\d+(?:\.\d+)?)\s*%` | % |
| 湿地保护率 | `湿地保护率[^\d%]{0,16}(\d+(?:\.\d+)?)\s*%` | % |

**扩展监控指标**：在 `validators.py` 的 `METRIC_PATTERNS` 字典中添加。

#### metric_year_mismatch（指标年份不匹配）

- **代码**：待实现
- **优先级**：P0
- **严重程度**：`warning`
- **检测逻辑**：指标数值对应的年份与规划基准年/目标年明显偏离

```python
# 在 validators.py 中添加新的验证函数
def _validate_custom_rule(drafts: list[ChapterDraft]) -> list[QualityFinding]:
    findings = []
    for draft in drafts:
        # 自定义检测逻辑
        if some_condition:
            findings.append(QualityFinding(
                severity="warning",
                code="custom_rule",
                message="检测到问题",
                location=draft.outline_id,
                suggestion="修复建议",
            ))
    return findings

# 在 validate_document() 中调用
def validate_document(outline, drafts):
    findings = []
    findings.extend(_validate_outline_coverage(outline, drafts))
    findings.extend(_validate_forbidden_phrases(drafts))
    findings.extend(_validate_evidence_required(drafts))
    findings.extend(_validate_metric_consistency(drafts))
    findings.extend(_validate_custom_rule(drafts))  # 新增
    return findings
```

## 四、质检报告格式

质检报告为 Markdown 格式，结构如下：

```markdown
# 质检报告

## 1. metric_conflict [error]
- 位置：2.2
- 问题：指标"森林覆盖率"在不同章节数值不一致：42.1% vs 43.0%
- 建议：核对统计年鉴，以最新调查数据为准，删除不一致的表述。

## 2. forbidden_phrase [warning]
- 位置：1.1
- 问题：发现不规范表述：据说
- 建议：改为可核验来源，例如文件名、统计年鉴或主管部门材料。

## 3. missing_section [warning]
- 位置：7.2
- 问题：缺少章节草稿：项目库
- 建议：生成或补录该章节后再做全文质检。
```

## 五、质量评分（规划中）

未来的质检结果将输出质量评分（0-100），权重如下：

| 类别 | 权重 | 计算方式 |
|------|------|----------|
| 来源完整性 | 30% | 有来源的数据点 / 总数据点 |
| 指标一致性 | 25% | 1 - 冲突指标数 / 总指标数 |
| 禁用词 | 15% | 1 - 禁用词数 / 总字数 * 1000 |
| 章节完整性 | 20% | 已完成章节数 / 总章节数 |
| 格式规范 | 10% | 1 - 格式问题数 / 总章节数 |
